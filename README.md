# グルメ記録アプリ（Webプロトタイプ）

「グルメ記録アプリ仕様書 v2」に基づくWebアプリプロトタイプ。
料理の写真を登録するだけで、自分だけのグルメアルバムが自動的に整理・記録される。

## 起動方法

Node.js / Python 不要。PowerShell製の簡易サーバーで動かす。

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ..\server.ps1
```

を実行し、ブラウザで **http://localhost:5959/** を開く。
（Claude Codeのプレビュー機能からは `.claude/launch.json` の `gourmet-app` 設定で起動）

初回はデータが空。一覧タブの「サンプルデータで試す」ボタンでデモデータを投入できる。

## スマホで使う（PWA対応）

アプリはPWA（Progressive Web App）対応済み。ホーム画面に追加すると全画面のアプリとして起動し、オフラインでも記録の閲覧ができる。

### 同じWi-Fi内のスマホからアクセスする

1. **初回のみ**: PCで管理者PowerShellを開き `setup-smartphone.ps1` を実行（ポート開放とファイアウォール設定）
2. `server.ps1` を起動（再起動）すると「Phone URL: http://192.168.x.x:5959/」が表示される
3. スマホ（同じWi-Fi）のブラウザでそのURLを開く

※ LAN経由（http）では閲覧・登録はできるが、ブラウザの制限によりホーム画面インストールとオフライン機能は使えない。

### 完全なアプリ体験（ホーム画面インストール・オフライン）にする

PWAのインストールにはhttpsが必要。`gourmet-app/` フォルダを無料の静的ホスティング
（GitHub Pages / Cloudflare Pages / Netlify など）にアップロードすると、
スマホで開いて「ホーム画面に追加」→ アプリとして起動できるようになる。
データは端末のブラウザ内に保存されるため、ホスティング側には何も保存されない。

- iPhone: Safariで開く → 共有ボタン → 「ホーム画面に追加」
- Android: Chromeで開く → メニュー → 「アプリをインストール」

## 構成

| ファイル | 役割 |
|---|---|
| `index.html` | 画面レイアウト（登録・地図・一覧・写真・統計の5タブ） |
| `css/style.css` | スタイル（モバイル対応レスポンシブ） |
| `js/store.js` | データ層。店舗(Shop)/訪問記録(Visit)の2層モデル（仕様書§2）。localStorage＋写真はIndexedDB |
| `js/api.js` | 外部API連携（Overpass/Nominatim）・EXIF解析・画像圧縮・ジャンル推定 |
| `js/register.js` | 登録フロー（仕様書§4）。フローA(GPSあり)/フローB(名前検索・地図指定) |
| `js/views.js` | 地図・一覧・写真・統計・ランキング・店舗詳細の描画 |
| `js/app.js` | タブ制御・共通イベント・サンプルデータ |

## 実装済み機能（仕様書v2との対応）

- **登録（§4）**: 写真選択 → EXIF解析（撮影日時・GPS）→ 周辺店舗候補（Overpass API）/ 名前検索（Nominatim）/ 地図タップ指定。登録済み店舗は候補の先頭に表示し、選択すると再訪として記録
- **自動入力（§1.1）**: 住所・都道府県・市区町村（逆ジオコーディング＋ISOコード変換）、最寄駅（Overpass railway=station）
- **AIによる料理ジャンル判定（§5）**: Anthropic APIキーを ⚙️ 設定で登録すると、料理写真をClaude（claude-opus-4-8）が解析し、料理ジャンル・店舗ジャンルを構造化出力で自動判定。確信度が低い場合は空欄にして利用者に選択を促す。キー未設定時はOSMのcuisineタグからの推定にフォールバック
- **データモデル（§2）**: 店舗と訪問記録を分離。訪問回数・平均評価・代表写真は集計値として算出
- **地図（§6)**: 評価別の色分きピン、クラスタリング、ポップアップ（写真・評価・コメント・詳細リンク）、現在地移動、訪問頻度ヒートマップ（§9統計）
- **一覧（§7）**: 分類（駅/都道府県/市区町村/料理ジャンル/評価）× 並び替え（新着/訪問日/訪問回数/評価/店名）
- **検索（§8）**: キーワード＋都道府県＋ジャンル＋評価＋お気に入りの複合絞り込み
- **写真（§9）**: 種別（料理/外観/店内/メニュー）、料理ジャンル別閲覧、長辺1600pxへ圧縮保存、拡大表示
- **評価（§10）**: 訪問ごとに★1〜5、後から編集可能、店舗は平均評価を表示
- **統計（§11相当）**: 総店舗数・総訪問回数・写真枚数・平均評価・年間/月別訪問・ジャンル割合・都道府県別・6種のランキング
- **お気に入り（§12）**: 店舗単位のON/OFF、一覧絞り込み
- **エッジケース（§16）**: 位置情報なし写真（主要フローとして検索/地図指定）、閉店管理、利用形態（店内/テイクアウト/デリバリー）、訪問日時の手動修正

## プロトタイプでの簡略化（本番実装との差分）

| 項目 | プロトタイプ | 本番想定（仕様書） |
|---|---|---|
| 料理ジャンルのAI判定 | 実装済み（Claude API・利用者のAPIキーをブラウザに保存して直接呼び出し） | サーバー経由でAPIキーを秘匿して呼び出し |
| データ保存 | ブラウザ内（localStorage/IndexedDB） | クラウド（§12・§14） |
| アカウント・認証（§3） | なし（単一利用者） | Apple/Googleサインイン |
| オフライン同期（§14.1） | ブラウザ内保存のため常にローカル | ローカル→クラウド自動同期 |
| 店舗検索API | Overpass/Nominatim/Photon（無料・キー不要）＋ Google Places（⚙️でキー設定時。OSM未登録の個人店もヒット） | サーバー経由でキーを秘匿 |

※ ブラウザのサイトデータを消去すると記録も消えるので注意。
